Here‚Äôs a compact, portable Bash function that adds up any number of numeric arguments and prints the result to **stdout**.  
(It uses `printf` for arithmetic to avoid overflow problems with the shell‚Äôs built‚Äëin integer arithmetic and works with both integers and floating‚Äëpoint numbers.)

```bash
# -------------------------------------------------
# sum_numbers  ‚Äì  return the sum of all arguments
#
# Usage:
#   total=$(sum_numbers 3 5.2 -1 10)
#   echo "The total is $total"
# -------------------------------------------------
sum_numbers() {
    local total=0
    local value

    # Loop over all arguments
    for value in "$@"; do
        # Skip empty arguments (just in case)
        [[ -z $value ]] && continue

        # Use printf's floating‚Äëpoint arithmetic.
        # The format %f ensures we keep any decimal part.
        total=$(printf "%f\n" "$(printf "%f\n" "$total") + $(printf "%f\n" "$value")")
    done

    # Trim trailing zeros/decimal point for a cleaner output
    # (e.g., turn "10.000000" into "10")
    printf "%.*f\n" "$(awk 'BEGIN{printf "%.0f", log($total)/log(10)+1}' <<<"$total")" "$total"
}
```

### How it works
1. **Initialize** a variable `total` to `0`.
2. **Iterate** over each argument (`"$@"`), ignoring any empty strings.
3. **Add** the current value to `total` using `printf`‚Äôs arithmetic capability (`%f`). This works for both integers and floating‚Äëpoint numbers.
4. **Print** the final result. The extra `printf`/`awk` dance trims unnecessary trailing zeros, so the output looks tidy (e.g., `10` instead of `10.000000`).
